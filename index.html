Prework Study Guide

HTML

  • !DOCTYPE html : This declaration tells the browser what document type it should expect when it is opened.
  • Inside HTML tag, attribute lang="en" will state what language web page will be in. In this example "en" is for English.
  • The head element contains information about the webpage called Metadata.
  • Inside the head tag the following metadata tags can be found: Title, Style, Base, Link, Meta and Script.
  • Title element defines title for the webpage. This is required and important for Search Engine Optimization (SEO)
  • Style element defines the CSS styles associated with a website.
  • Base element defines the base URL for a webpage.
  • Link element connects an external resource to the HTML document.
  • Meta element defines metadata such as the character set, description, keywords, author, and viewport.
  • Script element embeds client-side scripts in the HTML document.
  • The body element represents the visible content shown to the user.
  • Body element has many tags that can be used including h1 - h6 (level of heading), p (block of text), ul (unordered lists), ol (ordered lists), li (list items), img (info about images) which can include attributes ie. `src` (important - defines location) or `alt` (contains a text string that describes the appearance and functionality of the image), br (creates empty line or break between two blocks of content).
  • a href= - a : anchor, creates links to the same webpage or other webpages. Href: attribute points to the URL for the link
  • Footer element appears at the bottom of the page and usually contains the author, copyright, contact, sitemap, and navigation.

CSS

  • Inline CSS: style element directly in HTML file. Inline styles apply to a specific HTML tag, using a style attribute with a CSS rule to style a particular page element only. Useful for quick permanent changes, less flexible than external/internal style sheets because each inline style must be edited individually whenever you make a design change.
  • Internal Style Sheet: contains CSS rules for the webpage in the head section of the HTML file using `style` element. Rules only apply to that page and can configure CSS classes and IDs to style multiple elements in the page code.
  • External CSS Style Sheet: Preferred method because it allows developers to keep all CSS rules in a seperate sheet. File is linked with `link` element placed in HTMLs `head` element.
  • Declaration: contain two important components: the CSS property we want to apply and the value of the property.
  • Margin Property: property indicates how much space we want around the outside of an element. When set to 0, we are indicating that we want NO extra space around the element.
  • Padding Property: adds space around the content inside an element. When padding is increased, makes element bigger because the padding, while transparent, is located inside the element, whereas the margin is the transparent buffer outside an element.
  • Wild Card Symbol or * is used to apply a rule to all the elements visible on the page.
  • Two selectors can be combined into one CSS rule by seperating selectors with a comma.
  • Link element: creates connection to external file. Rel Attribute: specifies the relationship between the current document and the linked document resource. href attribute: specifies the location (URL) of the external resource.
  • Card: designates this as a class selector. Any HTML element assigned to the class, card, will gain these property assignments from the CSS.

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git checkout main: switches to main branch
  • git add -A: add command adds modifications in the current working branch to the staging area. The -A flag indicates that we want to add all changes. The staging area informs Git that the changes will be added in the next commit.
  • git commit -m "add starter code": commits staged files with a commit message that describes what the commit contains. -m flag to associate a message with our commit.
  • git pull origin main: git (identifies git command), pull (receive a branch's modifications into the local environment), origin ( indicates the source of the pull will be in the GitHub repo), main (indicates the branch).
  • It's important to always do a git pull to make sure that you have the latest version of the project on your local computer before you start to work.
  • git push origin feature/starter-code: the push command pushes the changes we have locally to our remote GitHub branch. You can create a new remote branch or add new changes to an existing branch.
